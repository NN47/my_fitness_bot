# Рефакторинг бота-ассистента

## Что было сделано

### ✅ Создана модульная структура проекта

Проект разбит на логические модули:

```
my_fitness_bot/
├── config.py              # Централизованная конфигурация
├── main.py                # Точка входа (новый)
├── bot.py                 # Старый монолитный файл (сохранён для справки)
├── database/
│   ├── __init__.py
│   ├── models.py          # SQLAlchemy модели
│   └── session.py         # Управление сессиями БД
├── handlers/
│   ├── __init__.py
│   ├── common.py          # Общие обработчики
│   ├── start.py           # Обработчик /start
│   ├── workouts.py        # Обработчики тренировок
│   └── meals.py           # Обработчики КБЖУ
├── services/
│   ├── __init__.py
│   ├── gemini_service.py  # Сервис Gemini API
│   └── nutrition_service.py # Сервис CalorieNinjas и Open Food Facts
├── states/
│   ├── __init__.py
│   └── user_states.py     # FSM состояния
└── utils/
    ├── __init__.py
    ├── keyboards.py        # Клавиатуры
    ├── formatters.py      # Функции форматирования
    ├── validators.py      # Валидаторы
    └── logging_config.py  # Настройка логирования
```

### ✅ Улучшения

1. **Централизованная конфигурация** (`config.py`)
   - Все настройки в одном месте
   - Легко менять без правки кода

2. **Управление БД** (`database/session.py`)
   - Контекстный менеджер для сессий
   - Автоматический commit/rollback
   - Инициализация БД в одном месте

3. **Логирование** (`utils/logging_config.py`)
   - Замена `print()` на структурированное логирование
   - Логи в файл и консоль
   - Настраиваемые уровни

4. **Сервисы** (`services/`)
   - Изоляция работы с внешними API
   - Легко тестировать и мокировать
   - Централизованная обработка ошибок

5. **FSM состояния** (`states/`)
   - Готова структура для замены `getattr/setattr`
   - Типизированные состояния

6. **Модульные обработчики** (`handlers/`)
   - Каждый модуль отвечает за свою область
   - Легко добавлять новые функции
   - Проще тестировать

## Как использовать новую структуру

### Запуск бота

```bash
python main.py
```

Вместо старого:
```bash
python bot.py
```

### Переменные окружения

Создайте `.env` файл:
```
API_TOKEN=your_telegram_bot_token
DATABASE_URL=sqlite:///fitness_bot.db
GEMINI_API_KEY=your_gemini_api_key
NUTRITION_API_KEY=your_calorieninjas_api_key
```
